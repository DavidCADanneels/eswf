Description

The zipchunk module provides functions for zlib-compressed byte sequence templates, allowing for ahead-of-time compression of fixed chunks. It is designed to be used within higher-level templating systems.

Data Types

sumtype()

sumtype() = adler32 | crc32 | none

template()

template() = [{hole, Key::term()} | {chunk, iodata()} | {zchunk, iodata(), Length::integer(), Chunksum::integer()}]

A template() is a sequence of rule tuples tagged by an atom. A hole rule indicates the location of a sequence of bytes that will be provided later. A chunk rule places a byte sequence into the resulting zlib stream's logical plaintext sequence. A zchunk rule places a byte sequence directly into the resulting zlib stream and provides the Length and Chunksum of the corresponding logical plaintext sequence.
